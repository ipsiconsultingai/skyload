# 생활기록부 데이터 명세서

## 개요

생활기록부(생기부) PDF 분석 결과를 기반으로 설계된 통합 데이터 모델.
PDF 업로드, 이미지 업로드, 직접 입력 3가지 방식 모두에서 최종적으로 저장되는 구조.

---

## 1. 출결상황 (Attendance)

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| 학년 | number (1-3) | O | |
| 수업일수 | number | O | |
| 결석일수 | number | O | 질병/미인정/기타 합산 |
| 지각 | number | O | 합산 |
| 조퇴 | number | O | 합산 |
| 결과 | number | O | 합산 |
| 특기사항 | string \| null | X | |

> PDF 원본에는 결석/지각/조퇴/결과 각각 질병/미인정/기타로 세분화되지만,
> 입시 분석에 핵심적인 것은 합산 수치. 직접입력 폼에서는 합산값만 받음.

## DB 저장 방식

하이브리드 방식: `records` 테이블에 메타정보 + `record_data` JSONB 컬럼에 전체 구조화 데이터.

- 섹션이 많고 과목 수가 가변적이라 정규화 시 11+ 테이블 필요
- AI 분석 시 전체 데이터를 한번에 읽으므로 JSONB가 효율적
- TypeScript의 `SchoolRecord` 인터페이스가 JSONB의 스키마 역할
