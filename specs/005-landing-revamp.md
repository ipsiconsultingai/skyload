# 랜딩 페이지 & 서비스 소개 페이지 리뉴얼

> 한 줄 요약: 랜딩 페이지와 서비스 소개(/about) 페이지의 섹션 구조를 재설계하고, "AI + 전문가 이중 검증" 차별점을 중심으로 전환 최적화된 카피를 작성한다.

## 개요

### 배경

- 현재 랜딩 페이지는 기본적인 구조(Hero → Services → Process → Pricing → Preview → Reviews → FAQ → CTA)를 갖추고 있으나, "AI + 전문가 이중 검증"이라는 핵심 차별점이 충분히 부각되지 않음
- Hero 섹션의 카피("합격에는 이유가 있습니다")가 서비스 본질을 즉시 전달하지 못함
- 서비스 소개 페이지(/about)가 기능 나열 위주로, Pain Point → Solution → Proof 흐름이 약함
- 신뢰 지표(분석 건수, 정확도, 전달 시간 등)가 분산되어 있어 임팩트가 떨어짐

### 목표

1. 방문자가 3초 이내에 "AI + 전문가가 생기부를 이중 검증해주는 서비스"임을 파악
2. Pain Point → Solution → Social Proof → CTA 전환 퍼널 최적화
3. 숫자/데이터 기반 신뢰감 강화
4. 기존에 잘 작동하는 섹션(Pricing, FAQ, Reviews)은 구조 유지하되 카피 다듬기

### 핵심 메시지

**"AI + 전문가 이중 검증"** — AI가 분석하고, 전문가가 검수합니다.

### 디자인 원칙

- 기존 CSS 변수/디자인 시스템 유지 (globals.css)
- CSS Modules 사용 (Tailwind X)
- FadeIn 컴포넌트 활용한 스크롤 애니메이션
- 모바일 퍼스트 반응형 (640px, 768px, 1024px)
- 이미지/영상은 placeholder로 처리

## Part 1: 랜딩 페이지 (/) 섹션 구조

### 섹션 구조 총괄

| 순서 | 섹션명 | 목적 | 전환 퍼널 단계 |
|------|--------|------|----------------|
| 1 | Hero | 핵심 가치 전달 + 첫 CTA | Attention |
| 2 | Trust Bar | 핵심 수치로 즉각 신뢰 형성 | Trust |
| 3 | Pain Point | 타겟의 고민 공감 | Interest |
| 4 | Solution (Difference) | AI+전문가 차별점 | Interest |
| 5 | Process | 이용 과정 설명 | Desire |
| 6 | Service Cards | 제공 서비스 상세 | Desire |
| 7 | Preview | 리포트 샘플 미리보기 | Desire |
| 8 | Pricing | 플랜 비교 + 구매 | Action |
| 9 | Reviews | 이용자 후기 | Trust/Action |
| 10 | FAQ | 구매 장벽 해소 | Action |
| 11 | Final CTA | 마지막 전환 유도 | Action |

### 1. HeroSection (전면 개편)

**목적**: 방문 즉시 서비스 핵심 가치 전달, 첫 번째 CTA 유도

**핵심 카피**:

- **배지**: `AI + 전문가 이중 검증 시스템` (ShieldCheck 아이콘)
- **헤드라인**: `생기부 분석, AI만으로는 부족합니다`
- **서브텍스트**: `AI가 정밀 분석하고, 입시 전문가가 한 줄 한 줄 검수합니다.` / `48시간 안에 완성되는 맞춤 리포트.`
- **CTA (주)**: `무료 진단 시작하기` + ArrowRight
- **신뢰 지표**: 3개 아이콘 배지 — `AI 정밀 분석` / `전문가 검수 포함` / `48시간 내 전달`

**비주얼 (우측 프리뷰 카드)**:
- 기존 scoreRing + bar chart UI 유지
- "AI 분석 완료 → 전문가 검수 완료" 2단계 배지 강조
- 프리뷰 카드 상단에 2단계 진행 시각화

**디자인 힌트**:
- 텍스트 중심 좌측 배치 + 우측에 리포트 UI 프리뷰
- 배경: 어두운 그라데이션(현재 유지) + 미묘한 그리드 패턴
- 뱃지에 ShieldCheck 아이콘으로 "검증" 뉘앙스

**현재 대비 변경점**:
- 헤드라인 변경: "합격에는 이유가 있습니다 / 우리는 그 이유를 만듭니다" → "생기부 분석, AI만으로는 부족합니다" (차별점 직접 전달)
- 서브텍스트에 "전문가 검수" 명시적 노출
- Social Proof를 "N명이 진단을 완료" → 3개 아이콘 배지로 변경
- CTA 텍스트 유지: "무료 진단 시작하기"

### 2. TrustSection (신규)

**목적**: Hero 직후 핵심 수치 4개로 즉각 신뢰 형성

**핵심 카피**:

| 수치 | 라벨 |
|------|------|
| `2,400+` | `누적 분석 건수` |
| `4.8/5.0` | `평균 만족도` |
| `100%` | `전문가 검수율` |
| `38시간` | `평균 리포트 전달` |

**디자인 힌트**:
- 가로 1줄 배치, 4칸 그리드
- 배경: 투명 또는 미묘한 구분선
- 수치는 큰 폰트(Bold) + 라벨은 작은 폰트(Muted)
- CountUp 애니메이션 (스크롤 진입 시 useIntersectionObserver 활용)
- Hero와 자연스럽게 연결되도록 간격 최소화

### 3. PainPointSection (신규)

**목적**: 타겟 학생/학부모의 실제 고민을 짚어 공감대 형성

**핵심 카피**:

- **레이블**: `Pain Points`
- **헤드라인**: `혹시 이런 고민, 하고 계신가요?`

**카드 구성 (4개)**:

| 아이콘 | 고민 제목 | 설명 |
|--------|----------|------|
| `HelpCircle` | `내 생기부, 객관적으로 어느 수준일까?` | `선생님 말씀만으로는 판단이 어렵고, 비교 기준이 없어 불안합니다.` |
| `AlertTriangle` | `남은 학기에 뭘 해야 할지 모르겠다` | `활동은 많이 했는데, 방향이 맞는 건지 확신이 없습니다.` |
| `Target` | `수시 지원, 어디에 써야 할까?` | `내 생기부 강점을 살릴 대학과 전형을 모르겠습니다.` |
| `Shield` | `AI 분석만으로 충분할까?` | `ChatGPT에 넣어봤지만 정확한 건지 의문이 듭니다.` |

**디자인 힌트**:
- 2x2 그리드 (모바일 1열)
- 각 카드: 아이콘 + 제목(Bold) + 설명(Muted)
- 부드러운 배경색(light gray/off-white)
- 마지막 카드("AI 분석만으로 충분할까?")에서 자연스럽게 다음 섹션(Difference)으로 연결

### 4. DualVerificationSection (핵심 차별화)

**목적**: Pain Point의 답으로 "AI + 전문가 이중 검증" 차별점 제시

**핵심 카피**:

- **레이블**: `Why SKYROAD`
- **헤드라인**: `왜 AI + 전문가인가?`
- **서브**: `SKYROAD는 AI 분석 후, 전임 입시 컨설턴트가 직접 검수합니다`

**2컬럼 비교** (기존 about 페이지 DifferenceSection 구조 활용):

| | AI만 사용하는 서비스 | SKYROAD (AI + 전문가) |
|---|---|---|
| 분석 방식 | 키워드 기반 단순 매칭 | 문장 맥락 + 교과 연계 + 성장 서사 정밀 판단 |
| 검증 | 할루시네이션 검증 없음 | 전임 컨설턴트가 사실관계, 해석 오류 직접 수정 |
| 전략 | 일반적인 대학 추천 | 최신 입시 데이터 기반 맞춤 전략 |
| 품질 | 정확도 편차가 큰 결과 | 이중 검수로 일관된 분석 품질 보장 |

**디자인 힌트**:
- 좌: "AI만 사용" 카드 — 회색/흐린 톤 + bullet 마크
- 우: "AI + 전문가" 카드 — 브랜드 컬러 강조 + Check 마크 + "추천" 뱃지
- [이미지: 전문가가 리포트를 검토하는 모습 일러스트 또는 "AI → 전문가 검수" 플로우 다이어그램]

**현재 대비 변경점**:
- 기존 about 페이지 DifferenceSection 구조/카피를 랜딩 페이지로 이동
- 헤드라인 변경: "AI 분석만으로는 부족합니다" → "왜 AI + 전문가인가?"

### 5. ProcessSection (개선)

**목적**: 이용 절차를 3단계로 간결하게 보여줘 진입 장벽 낮추기

**핵심 카피**:

- **레이블**: `How it works`
- **헤드라인**: `3단계로 완성되는 전문 리포트`
- **서브**: `업로드부터 리포트 수령까지, 간단합니다`

| 단계 | 제목 | 설명 | 세부사항 |
|------|------|------|----------|
| STEP 01 | `생기부 업로드` | `PDF 또는 텍스트를 업로드하세요. 1분이면 충분합니다.` | 한글 HWP 파일 지원 / NEIS 생기부 자동 파싱 |
| STEP 02 | `AI 정밀 분석` | `최신 AI가 세특, 창체, 내신, 모의고사 등 생기부 전 영역을 분석합니다.` | 5개 핵심 지표 분석 / 합격 사례 데이터 기반 비교 |
| STEP 03 | `전문가 검수 & 리포트 전달` | `전임 입시 컨설턴트가 AI 결과를 직접 재검토합니다. 48시간 내 리포트 전달.` | 10~25페이지 심층 분석 / 맞춤 전략 & 보완점 제시 |

**디자인 힌트**:
- 세로 타임라인 레이아웃 (현재 구조 유지하되 비주얼 개선)
- 3번째 단계에 "전문가 검수" 강조 뱃지 추가
- [이미지: 각 단계별 UI 스크린샷 또는 아이콘]

**현재 대비 변경점**:
- 카피 거의 동일 (현재 잘 작성됨)
- 서브 수정: "3단계만으로 전문 수준의 생기부 분석 리포트를 받아보세요" → "업로드부터 리포트 수령까지, 간단합니다"
- STEP 03 설명 미세 수정: "수정합니다" → "재검토합니다"

**현재 대비 변경점**:

**비주얼**:

### 8. PricingSection (구조 유지, 디자인 개선)

**목적**: 플랜 비교를 통한 구매 전환

**핵심 카피**:

- **헤드라인**: `나에게 맞는 플랜을 선택하세요`

| 플랜 | 가격 | 분량 | 한줄 포인트 |
|------|------|------|-------------|
| Lite Report (기본 진단형) | 59,000원 | 10~12p | `내 생기부 현재 위치, 빠르게 파악하기` |
| Standard Report (실전 전략형) | 99,000원 | 15~18p | `합격 가능성 분석 + 지원 전략까지` |
| Premium Report (프리미엄 설계형) | 199,000원 | 20~25p | `오프라인 컨설팅 수준, 온라인 가격으로` |

**디자인 힌트**:
- 기존 PricingSection 구조 그대로 유지 (완성도 높음)
- Standard에 "가장 인기" 뱃지 유지
- "전임 컨설턴트 검수 포함" 뱃지 유지
- 카드 디자인 모던화 고려 (그라데이션 보더, 호버 효과 강화)

**현재 대비 변경점**:
- 카피 변경 없음 (현재 완성도 높음)
- 디자인만 소폭 모던화
