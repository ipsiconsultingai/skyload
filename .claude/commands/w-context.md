현재 작업 중인 폴더에 CONTEXT.md 파일을 생성하거나 업데이트합니다.

## 목적

AI가 해당 모듈의 맥락을 빠르게 파악할 수 있도록 자연어로 된 문서를 작성합니다. 코드 주석 대신 이 문서를 통해 설계 의도와 사용 방법을 전달합니다.

## 실행 방법

1. 현재 작업 중인 폴더 경로 확인
2. 해당 폴더의 모든 파일 분석
3. CONTEXT.md 파일 존재 여부 확인
   - 없으면: 새로 생성
   - 있으면: 현재 코드와 비교하여 변경된 내용 반영

## 업데이트 시 규칙

기존 CONTEXT.md가 있는 경우:

1. 현재 폴더의 코드를 분석하여 변경사항 파악
2. 기존 문서의 구조는 유지하면서 내용 업데이트
3. 새로 추가된 파일이나 기능은 문서에 반영
4. 삭제된 파일이나 기능은 문서에서 제거
5. 변경된 설계 결정이나 사용 패턴 업데이트

## CONTEXT.md 작성 규칙

### 필수 포함 내용

1. **이 모듈이 하는 일**: 한 문장으로 핵심 기능 설명
2. **파일 구조와 역할**: 각 파일이 담당하는 역할
3. **핵심 설계 결정**: 왜 이렇게 구현했는지, 주요 트레이드오프
4. **사용 패턴**: 실제 사용 예시 코드
5. **확장 시 고려사항**: 기능 추가 시 알아야 할 점

### 작성 원칙

- 코드를 읽지 않아도 이해할 수 있어야 함
- 핵심만 간결하게 작성
- 구현 세부사항보다 의도와 맥락에 집중
- 마크다운 형식 사용

## 파일 생성 위치

작업 중인 폴더 내부에 `CONTEXT.md` 파일로 생성합니다.

예시:

- `libs/logger/` 작업 시 → `libs/logger/CONTEXT.md`
- `libs/api/` 작업 시 → `libs/api/CONTEXT.md`
- `components/Button/` 작업 시 → `components/Button/CONTEXT.md`

## 출력 형식

```markdown
# [모듈명] 컨텍스트

이 문서는 AI가 [모듈명] 모듈의 설계 의도와 사용 방법을 빠르게 파악할 수 있도록 작성되었습니다.

## 이 모듈이 하는 일

[한 문장 설명]

## 파일 구조와 역할

- `파일명.ts`: 역할 설명
- ...

## 핵심 설계 결정

1. **결정 1**: 설명
2. **결정 2**: 설명

## 사용 패턴

\`\`\`typescript
// 사용 예시
\`\`\`

## 확장 시 고려사항

[확장 시 알아야 할 내용]
```
