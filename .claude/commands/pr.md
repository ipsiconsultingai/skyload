현재 변경사항을 커밋하고 PR을 생성합니다.

## 실행 순서

1. `git status && git branch --show-current` 실행
2. 브랜치가 feature/fix/hotfix로 시작하지 않으면 변경 내용을 분석하여 적절한 브랜치 자동 생성
3. 변경사항을 논리적 단위로 분석하여 적절히 커밋 분리
4. `git push -u origin <브랜치>`
5. 해당 브랜치에 PR이 없으면 `gh pr create --assignee @me`, 있으면 PR 링크만 출력
6. PR 링크 출력

## 브랜치 타입

| 타입       | 용도             | 예시                 |
| ---------- | ---------------- | -------------------- |
| `feature/` | 새로운 기능 추가 | `feature/explore`    |
| `fix/`     | 버그 수정        | `fix/link-drag-drop` |
| `hotfix/`  | 긴급 버그 수정   | `hotfix/login-error` |

## 커밋 규칙

변경사항을 논리적 단위로 분리하여 커밋합니다:

- **기능 단위**: 하나의 기능이 완성되는 단위로 커밋
- **레이어 단위**: API, UI, 타입 등 레이어별로 분리 가능
- **의존성 순서**: 다른 코드가 의존하는 코드를 먼저 커밋

### 커밋 분리 예시

```
1. [탐색] 타입 정의 및 API 함수 추가
2. [탐색] Hooks 추가
3. [탐색] UI 컴포넌트 추가
4. [탐색] 페이지 레이아웃 및 라우팅 추가
```

### 커밋 메시지 형식

```
[카테고리] 작업 제목

작업 설명 (복잡한 변경일 경우에만)
```

### 카테고리

보관함, 탐색, 링크, 폴더, 휴지통, 상세보기, 요약, 공통, 랜딩, 검색, DnD 등 (작업 대상 모듈/기능명 사용)

### 커밋 예시

**간단한 커밋:**

```
[링크] 링크 추가 모달 UI 구현
```

**상세 설명이 필요한 커밋:**

```
[보관함] 드래그 앤 드롭 로직 개선 및 충돌 감지 최적화

주요 변경사항:
- 폴더 간 링크 이동 시 충돌 감지 알고리즘 개선
- 드래그 중 스크롤 동작 최적화
- 멀티셀렉트 상태에서 드래그 앤 드롭 지원
```

## PR 규칙

### PR 제목

```
[카테고리] 작업명
```

### PR 본문

- 핵심 변경사항만 간결하게 요약
- 리뷰어가 알아야 할 중요한 맥락만 포함
- 커밋 내역을 나열하지 않음
- 가독성 최우선

```markdown
## 요약

[작업 목적과 주요 변경사항을 2-3문장으로 설명]

## 주요 변경사항

- [변경사항 1]
- [변경사항 2]

## 참고사항 (선택)

[리뷰어가 알아야 할 추가 컨텍스트]
```
